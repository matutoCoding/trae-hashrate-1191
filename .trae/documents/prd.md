## 1. 产品概述

游艇租赁出海管理系统是一套纯前端应用，面向游艇租赁公司运营人员，提供从游艇建档、预订排期、冲突校验、时段计费到账单生成的全流程管理能力。核心解决同一游艇同一时段的排期冲突问题，并实现按高峰/平峰多档费率跨档分段计费，确保资源不重复占用、费用计算精确透明。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 说明 | 核心权限 |
|------|------|----------|
| 运营管理员 | 系统主要操作者 | 游艇建档、预订管理、费率维护、账单生成、审批出海报备 |
| 审批员 | 出海报备审批 | 审批/驳回出海报备申请 |

### 2.2 功能模块

1. **仪表盘**：今日出海概览、待审批提醒、营收统计、近期预订
2. **游艇管理**：游艇建档、游艇列表、游艇详情与排期日历
3. **预订排期**：新建预订、排期日历/列表视图、冲突校验提示、退订与时段释放
4. **费率管理**：时段费率表 CRUD、高峰/平峰/低谷档位定义、费率切换点配置
5. **账单中心**：跨档分段计费明细、账单生成与查看、出海报备审批流程

## 3. 核心流程

### 3.1 预订与冲突校验流程

用户选择游艇和时段后，系统自动与该游艇已有有效预订进行时段重叠校验。若存在冲突则阻止提交并提示冲突详情；若无冲突则创建预订并锁定时段。退订时释放已占时段，允许新预订使用。

```mermaid
flowchart TD
    "A[选择游艇与时段]" --> "B[时段重叠校验]"
    "B" --> "C{存在冲突?}"
    "C" --> "是" --> "D[提示冲突详情，阻止提交]"
    "C" --> "否" --> "E[创建预订，锁定时段]"
    "E" --> "F[提交出海报备审批]"
    "F" --> "G{审批通过?}"
    "G" --> "是" --> "H[生成账单]"
    "G" --> "否" --> "I[驳回，可修改后重新提交]"
    "J[退订操作]" --> "K[释放已占时段]"
    "K" --> "L[时段可被新预订使用]"
```

### 3.2 跨档分段计费流程

预订时段可能跨越多个费率档位（如 10:00-15:00 跨越平峰→高峰→平峰），系统将总时长按费率切换点拆分为多段，每段按对应费率计算小计，最终合计为总金额。

```mermaid
flowchart TD
    "A[获取预订起止时间]" --> "B[查询费率表]"
    "B" --> "C[识别跨费率切换点]"
    "C" --> "D[按切换点拆分时段]"
    "D" --> "E[逐段计算：时长 × 费率]"
    "E" --> "F[合计各段金额]"
    "F" --> "G[生成账单明细]"
```

## 4. 用户界面设计

### 4.1 设计风格

- **主题色调**：深海蓝（#0C2340）为主色，金色（#C9A96E）为强调色，营造高端游艇航海氛围
- **辅助色**：浪花白（#F5F7FA）、深海灰（#2D3748）、碧海青（#0EA5E9）
- **按钮风格**：圆角微弧（rounded-lg），主按钮深蓝底金色文字，次按钮描边风格
- **字体**：标题使用 Playfair Display，正文使用 Noto Sans SC
- **布局风格**：左侧导航栏 + 右侧内容区，卡片式模块布局
- **图标风格**：lucide-react 线性图标，搭配航海主题意象
- **动效**：卡片悬浮微升阴影、页面切换淡入、数据加载骨架屏

### 4.3 响应式设计

- 桌面优先设计，侧边栏导航在大屏常驻
- 平板端侧边栏折叠为图标模式
- 移动端切换为底部标签栏，内容区单列堆叠

### 4.4 设计情绪

- **航海奢华感**：深海蓝与金色的经典搭配，传达高端、信赖、海洋气息
- **专业高效**：信息层次清晰，操作路径短，数据可视化直观
- **动态感**：微动画营造波浪流动感，让界面不显呆板
